--- layout: default --- {% comment %} _layouts/specialty-vertical.html Shared layout for every individual dental/medical specialty page (e.g. dental/orthodontic.md, medical/dermatology.md) — which vertical it's in comes from the `vertical` front-matter default set per-folder in _config.yml, and only affects breadcrumb/active-nav styling. Most specialty pages use the original, simpler front matter (headline/ subheadline/pain_point/services_intro). A page can opt into a richer, longer-form template instead by setting `hero:` (an object) — when set, it replaces the plain audience hero with a split hero + image, and unlocks several additional optional sections below. Setting `hero` also hides the legacy `pain_point` paragraph and the "Our Work" portfolio placeholder, since the richer template supersedes them. Front matter: title: SEO title, e.g. "Orthodontic Website Design in Los Angeles" description: meta description specialty: e.g. "Orthodontics" (used for breadcrumb text) vertical: "dental" or "medical" (set via _config.yml defaults) hide_breadcrumb: set true to hide the "← For Dental/Medical" link -- legacy fields (used when `hero` is not set) -- headline: hero headline, e.g. "Website Design for Orthodontists in Los Angeles" subheadline: hero subtitle pain_point: a short paragraph naming the specialty's specific problem -- richer template (used when `hero` is set) -- hero: { title, subtitle, text, image, image_alt, primary_cta_text, primary_cta_url, secondary_cta_text, secondary_cta_url } value_prop: { eyebrow, title, text } stat_cards: { items: [{ number, label, text }] } — a 3-up stat strip where each stat also gets a short explanatory paragraph feature: { title, paragraphs: [...], checklist_title, checklist: [...strings], image, image_alt, cta_text, cta_url } — image-left, copy-right faq: { title, items: [{ question, answer }] } -- shared by both templates -- services_intro_title: heading above the services-in-context grid services_intro: paragraph under that heading cta: { title, text, primary_text, primary_url, secondary_text, secondary_url } {% endcomment %} {% if page.hero %}
